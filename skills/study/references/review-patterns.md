# Review Patterns

Use this reference when closing a study step or revising a plan.

## End-Of-Step Review

Include these four elements:

- what the user should now be able to do
- the main mistake or misconception to watch
- the next best action
- a short control prompt such as `continue`, `too hard`, `too easy`, or `I am stuck`

## When The User Struggles

- shrink the task
- restate the goal more concretely
- switch from abstract explanation to one worked example
- ask for one smaller piece of evidence

## When The User Is Ahead

- increase difficulty through transfer
- shorten explanation and increase practice
- collapse trivial substeps

## Replanning Triggers

Replan when:

- the user goal changes
- the deadline changes
- the baseline was misread
- the current stage is clearly too hard or too easy
- the user wants a different learning mode
