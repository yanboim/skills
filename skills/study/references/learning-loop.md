# Learning Loop

Use this reference when you need the default study-session flow.

## Default Loop

1. clarify the concrete learning target
2. confirm starting level with a light diagnostic
3. define the current stage goal
4. teach the smallest useful concept slice
5. ask for active practice
6. review the user's response
7. adjust pace, difficulty, or sequence

## Planning Rules

- Prefer 3 stages for a short plan.
- Keep stage names outcome-based, not chapter-based.
- Give each stage one completion signal.
- Sequence by dependency, not by completeness.

## Turn Rules

- Prefer one exercise per turn.
- Prefer one core correction over many minor notes.
- Prefer reducing scope when the user is stuck.
- Prefer harder transfer tasks when the user is cruising.

## Completion Signals

Good stage completion signals include:

- explain the concept accurately in plain language
- solve a representative beginner problem
- correct a flawed example
- apply the concept to a new case
- produce a small artifact that uses the concept
