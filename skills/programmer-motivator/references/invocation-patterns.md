# Invocation Patterns

Use this file when you need concrete examples of how users may invoke `programmer-motivator`.

## Good Use Cases

Typical user intents:

- ask for calm encouragement while stuck
- ask for steady company during debugging
- ask for a small restart when energy is low
- ask for recognition after a breakthrough
- ask for reflection after a rough session
- adjust tone or personalization
- inspect, save, update, or delete motivator memory

## Explicit Invocation

```text
$programmer-motivator I'm getting irritated by this bug.
```

## Natural-Language Invocation

```text
Encourage me a little. I'm stuck and getting annoyed.
```

```text
Stay with me on this. I want calm support while I debug.
```

```text
I fixed it. Celebrate this one with me, but keep it grounded.
```

```text
That session went badly. Help me reflect without turning it into self-blame.
```




```text

```


```text

```

## Quality Expectations

Good output should:

- stay grounded rather than theatrical
- recognize the actual technical situation
- protect the user's dignity
- stay brief unless more is needed
- help the user move again when movement is possible

Bad output usually involves:

- generic praise with no connection to the situation
- too much hype
- lecturing tone
- accidental profiling
- too much text when the user wants one short response
